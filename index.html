LinkedList源码分析

发表于 2023-04-10 更新于 2023-08-21 分类于 Java集合

LinkedList

  • 基于双向链表实现
  • 一般是不会使用到 LinkedList 的，需要用到 LinkedList 的场景几乎都可以使用 ArrayList 来代替（LinkedList的作者也不用）
  • 不要下意识地认为 LinkedList 作为链表就最适合元素增删的场景。实际上，LinkedList 仅仅在头尾插入或者删除元素的时候时间复杂度近似 O(1)，其他情况增删元素的平均时间复杂度都是 O(n)
  • 指定位置插入/删除时，需要先找到插入/删除的位置（O(n/2)），再修改指定节点的指针完成插入/删除 => O(n)

适配器模式：将不兼容的接口转换为可兼容的接口

什么是适配器模式？

适配器模式是一种结构型设计模式，用于将一个类的接口转换为客户端所期望的另一个接口。它允许不兼容的类能够合作，使得原本由于接口不匹配而无法工作的类能够一起工作。

适配器模式通过创建一个适配器类，将原本不兼容的接口转换为目标接口，从而使得客户端能够使用目标接口来调用适配器类。适配器类实现了目标接口，并在内部持有一个原本不兼容的类的实例，通过适配器类的方法来调用原本不兼容类的方法。

工厂模式：简化对象的创建过程

介绍

在软件开发中，对象的创建是一个常见的操作。通常情况下，我们可以直接使用 new 关键字来创建对象，但是在某些情况下，对象的创建过程可能会比较复杂，涉及到多个步骤或者依赖关系。这时候，使用工厂模式可以帮助我们简化对象的创建过程。

工厂模式是一种创建型设计模式，它提供了一种封装对象创建过程的方式，使得客户端代码不需要直接依赖具体的类来创建对象。通过使用工厂模式，我们可以将对象的创建逻辑抽象出来，使得客户端代码只需要与工厂接口进行交互，而不需要关心具体的对象创建细节。

门面模式：简化复杂系统的接口调用

什么是门面模式？

门面模式（Facade Pattern）是一种结构型设计模式，它提供了一个简单的接口，用于访问复杂子系统中的一组接口。门面模式通过封装子系统的复杂性，提供了一个更简单、更易于使用的接口给客户端。

  • 门面类：封装了子系统的一组接口，并提供一个简单的接口给客户端使用
  • 子系统类：一个或多个，实现子系统的各个接口和功能
